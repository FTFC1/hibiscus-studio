# 6. Contributing

We welcome contributions to improve this tool. Please follow these guidelines to ensure a smooth process.

## Reporting Bugs

*   If you find a bug, please create an issue in the GitHub repository.
*   Describe the issue in detail, including the steps to reproduce it, the expected behavior, and the actual behavior.
*   Include screenshots or error messages if possible.

## Proposing Enhancements

*   For new features or enhancements, please also open an issue to start a discussion before you begin working on the code.
*   This allows the team to align on the feature and ensure it fits within the project's goals.

## Development Workflow

1.  Create a new branch for your feature or bugfix.
2.  Write clean, readable code that follows the existing style and conventions of the project.
3.  Ensure all existing tests pass, and add new tests for your changes.
4.  **Update the documentation in the `/documentation` directory to reflect any changes you make.** This is a critical step.
5.  Submit a pull request with a clear description of your changes.
