Self-Serve Studio Booking Platform

Client: UK-based self-serve event studio Timeline: January 2026 (10 days) Status: Live production system

The Problem

Client ran self-serve studio bookings entirely manually:

  • Customer inquiries via Instagram/TikTok DMs
  • Acuity calendar for scheduling
  • Manual invoice generation (2 PDFs per booking: hire deposit + damage deposit)
  • Spreadsheet tracking for payments and availability
  • Manual email confirmations
  • Virtual assistant in different timezone (Thailand/UK coordination)
  • Monthly subscription cost for invoicing tool
Pain Points:
  • Time-consuming manual process for every booking
  • Human error in invoice calculations and email sending
  • Timezone coordination issues with VA
  • Manual payment confirmation (no automation)
  • Double-booking risk
  • Owner spending admin hours instead of growing business
  • Ongoing tool subscription costs
Business Impact: Manual workflow preventing owner from scaling. Operational overhead tied up resources. Monthly invoicing tool costs. No systematic way to gather client feedback on bookings.

The Solution

1. Automated Invoice System (Phase 1, Days 1-2)

Built PDF generation system with all business logic embedded:

  • Dynamic pricing calculation (£345-£885 based on event type)
  • 50% deposit logic automated
  • Professional formatting with venue branding
  • Dual invoice generation (hire + damage deposit)
  • Eliminated monthly invoicing tool subscription

2. Custom Booking Funnel (Phase 2, Days 3-5)

Replaced Acuity dependency with custom booking flow:

  • Direct booking interface (no third-party tool)
  • Instant invoice generation on booking
  • Automated email delivery via Resend API
  • Progressive enhancement (works without JavaScript)
  • Mobile-responsive design

3. Calendar Backend Integration (Phase 3, Days 5-7)

Synchronized calendar system across platforms:

  • Old Google Calendar + new Google Calendar → unified system
  • Real-time availability checking
  • Automatic conflict detection
  • Timezone handling (UK/Thailand coordination)

4. Admin Dashboard (Phase 4, Days 7-9)

Control panel with email-based authentication:

  • Email + OTP access (no password management)
  • Cloudflare Access security layer
  • Confirm/cancel/reschedule actions
  • Real-time calendar updates
  • Booking status tracking

5. Production Polish (Phase 5, Days 9-10)

  • Terminology updates (deposit → reservation fee)
  • Cancellation policy enforcement
  • FAQ integration
  • 100% of owner feedback incorporated same-day

Technical Stack

Backend: Cloudflare Workers (serverless, edge deployment)
Data: Cloudflare KV storage
Calendar: Google Calendar API (service accounts, dual-calendar sync)
PDF: pdf-lib (invoice generation)
Email: Resend API (automated delivery)
Security: Cloudflare Access (Zero Trust, email-based auth)
Frontend: Vanilla JavaScript (progressive enhancement)

Key Technical Decisions

Voice-Flow Decision Making

Used Telegram voice notes for rapid decision cycles. Technical discussions happened in real-time audio, not lengthy text threads. Enabled sub-24-hour decision-to-implementation cycles.

Cloudflare Workers Over VPS

Zero cold-start latency, global edge deployment, built-in security. Decision made in single voice call. No server maintenance overhead.

Progressive Enhancement

Forms work without JavaScript. System accessible across all devices and connection speeds. Booking flow doesn't break if JS fails.

Service Account Auth

No user OAuth flow for calendar access. Simpler VA handoff. Removed authentication friction for operations team.
